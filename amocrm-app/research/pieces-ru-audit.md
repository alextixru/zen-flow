# Аудит каталога pieces под русскоязычную экосистему (2026)

Контекст: Dzen.Team продаёт self-hosted автоматизацию клиентам amoCRM в России. Задача — понять, какие из ~750 pieces форка Activepieces реально работают/нужны в РФ в 2026 году, какие мертвы из-за санкций/блокировок/ухода вендоров, и чего не хватает для реальных сценариев RU SMB.

## Метод

1. Полный список: `packages/pieces/core/` (27 шт.) + `packages/pieces/community/` (722 шт.) = **749 pieces**.
2. Каждый piece отнесён к одной из категорий (A/B/C/D). Для однозначных случаев (общеизвестные core-утилиты, длинный хвост нишевых зарубежных SaaS без RU-специфики) даётся групповая пометка без построчного обоснования — иначе отчёт превращается в 700 одинаковых строк «работает технически, спроса нет». Обоснование по одной строке дано только для неочевидных/спорных случаев.
3. Для спорных статусов блокировки (Notion, Airtable, HubSpot, Atlassian, Slack, Discord, Figma, Mailchimp, AWS, Twilio, Binance, Pinterest) — проверено через веб-поиск на актуальные новости 2024–2026, а не только по памяти о 2022 годе.

## B. Работает, нишевое (~535 pieces)

Дефолтный бакет: длинный хвост зарубежных SaaS, по которым явных RU-специфичных блокировок не найдено, но и типичный клиент amoCRM (малый/средний бизнес в РФ) их не выбирает — либо инструмент слишком нишевой везде (десятки узкоспециализированных AI writing/AI voice/AI avatar стартапов), либо это конкурирующий продукт в категории, где у RU-бизнеса уже есть свой стек (иностранные CRM-конкуренты вроде `attio`, `close`, `copper`, `folk`, `streak`, `twenty`, `vtiger`; иностранный HR — `bamboohr`, `greenhouse`, `lever`, `ashby`, `workable`; иностранные helpdesk — `freshdesk`, `intercom`, `front`, `gorgias`, `crisp`; no-code/AI long-tail — сотни пieces вида `*-ai`, `*gpt*`, `*bot*` от небольших стартапов).

Логика: не выдумывать блокировку там, где её нет, но и не продвигать то, на что нет реального спроса. Полный список этих ~535 имён не дублируется построчно в этом отчёте — он совпадает с оставшейся частью `ls packages/pieces/community/` за вычетом списков A/C/D выше; при необходимости пришлю как приложение/CSV.

## Чего не хватает для RU-экосистемы (приоритеты)

1. **Мессенджер MAX (VK)** — государственно продвигаемая замена WhatsApp/Telegram, 35+ млн пользователей за 2025 год, с 2025–2026 обязателен для ряда категорий бизнеса при общении с потребителями. Готовые коннекторы уже есть у конкурентов (Wazzup, Chat2Desk, Radist.Online) — самый явный и срочный гэп.
2. **Телефония РФ**: Mango Office, UIS, Sipuni, Zadarma, Beeline Облачная АТС — critical, т.к. весь западный VoIP-кластер (см. C, кластер 8) для РФ номеров бесполезен, а click-to-call/запись звонков — топовый запрос клиентов amoCRM.
3. **Платежи РФ**: ЮKassa (ЮMoney), Robokassa, CloudPayments, Т-Касса (Тинькофф) — все иностранные платёжные pieces мертвы для РФ (кластер 5).
4. **Яндекс-экосистема**: Яндекс.Метрика, Яндекс.Директ (лиды с рекламы), Яндекс.Диск, Яндекс 360 для бизнеса — доминирующая RU-аналитика/реклама, прямой аналог отсутствующего/ограниченного Google Ads.
5. **1С / МойСклад** — синхронизация заказов/остатков/бухгалтерии, у МойСклад есть открытый публичный API, очень частый сценарий интеграции с amoCRM у RU SMB.
6. **GetCourse** — доминирующая RU-платформа для инфобизнеса/онлайн-курсов, огромное пересечение аудитории с amoCRM-клиентами.
7. **SMS-провайдеры РФ**: SMSC.ru, SMS.ru, P1SMS — западные SMS-гейтвеи не доставляют на RU-номера надёжно.
8. **Маркетплейсы**: Ozon Seller API, Wildberries API — синхронизация заказов в CRM, растущий спрос от RU SMB-продавцов.
9. **VK (ВКонтакте)** — лиды из VK Рекламы/сообщений сообществ, замена заблокированным Facebook/Instagram-лидам.
10. **Email-рассылки РФ**: Unisender, Sendsay, DashaMail, Mindbox — лучше доставляют на Mail.ru/Yandex, чем заблокированный Mailchimp.
11. **КЭДО/документооборот**: Диадок (СКБ Контур), СБИС, Астрал — замена юридически недействительной иностранной e-подписи (кластер 7).
12. **Доски задач (замена Trello/Jira)**: YouGile, Kaiten, Weeek — нейтральная замена заблокированному Atlassian-стеку (намеренно не Bitrix24 — это прямой конкурент amoCRM).
13. **Tilda** — доминирующий RU лендинг-билдер; частично уже закрывается через `webhook`/`http`, но нативный piece снизил бы трение настройки.
