# Установка / обновление виджета на аккаунте amo (ручная операция)

Загрузка архива — операция человека (онбординг/деплой), НЕ часть цикла разработки.
Агент собирает `amocrm-app/widget/dzenflow-widget.zip` (`./build.sh`) и пишет в activity
«ожидает ручной загрузки» — дальше по этой инструкции, занимает ~2 минуты.
Источник флоу: `~/amoai/DEPLOY.md` §2 (проверен живьём 2026-07-03) + `WIDGET_DEV_GUIDE` §7.

## Первичная установка (интеграции ещё нет)

1. Залогиниться на `dzenteamdev.amocrm.ru` под админом.
2. Левое меню → **амоМаркет**.
3. Справа сверху меню «⋯» → **«Создать интеграцию»** (или вкладка **«Мое приложение»**).
   Это ПРИВАТНАЯ интеграция — без модерации, активна сразу после загрузки.
4. В форме — кнопка **«Загрузить архив виджета»** → выбрать `dzenflow-widget.zip`.
5. Заполнить обязательные поля формы (название/описание — что угодно, это приватка),
   сохранить. Ошибки манифеста amo покажет модалкой с точным текстом — если ругается,
   текст ошибки отдать агенту.
6. После установки: амоМаркет → **Установленные** → блок «Приватные интеграции» —
   карточка виджета там. Открыть её и записать **widget_code** (виден в url/карточке) —
   он нужен агенту (activity W001).

## Обновление кода (интеграция уже установлена)

1. **Поднять `version` в `widget/manifest.json`** и пересобрать zip (`./build.sh`) —
   ОБЯЗАТЕЛЬНО: amo кэширует script.js по `?v=<version>`, без бампа клиенты
   останутся на старом коде.
2. амоМаркет → **Установленные** → скролл до **«Приватные интеграции»** → клик по карточке.
3. В модалке — **«Редактировать»** → форма «Редактирование интеграции».
4. **«Загрузить архив виджета»** → выбрать новый zip. (Инпут файла:
   `input#input-upload-archive`, скрыт стилями — в обычном браузере кнопка видна и так.)
5. Кнопка сохранения сменит подпись на **«Загрузить интеграцию»** — нажать;
   модалка вернётся на «Описание» = успех.
6. Проверка: жёсткий reload страницы amo (Cmd+Shift+R); свежесть бандла —
   в DevTools-консоли `fetch('/upl/<widget_code>/<N>/widget/script.js?v='+Date.now(),
   {cache:'no-store'})` и поиск строкового маркера из нового кода.

## Частые грабли

- «"settings" field is required» — в манифесте обязателен блок `settings` минимум с одним полем.
- Картинки: точные размеры (logo_main 400×272, logo_small 108×108, logo 130×100,
  logo_medium 240×84, logo_min 84×84), каждая <300КБ — иначе валидация.
- `manifest.json` должен лежать в КОРНЕ архива (не в подпапке), без `.DS_Store`.
- Линтер загрузчика режет `console.*`/`alert`/`confirm`/`prompt` в script.js —
  укажет строки в модалке ошибки.
- Настройки виджета, хранимые amo, стираются при деактивации/реактивации интеграции.
